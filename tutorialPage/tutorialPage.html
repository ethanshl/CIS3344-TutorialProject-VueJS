Vue.js 101

Introduction

Vue is is a progressive framework for building user interfaces. Compared to most inflexible frameworks, Vue was built to be incrementally adoptable. At its core Vue is focused on the view layer only, and is easy to pick up and integrate with other libraries or existing projects. That is not all, Vue is capable of powering sophisticated Single-Page Applications when used in combination with modern tooling and supporting libraries. In this tutorial, the version of Vue that we are going to use is 2.0; and we are not going to cover vue-cli like we did for the project here.



Getting Started

First, we need an IDE to work on Vue. VisualStudio is fine, but we may need to install a bunch of extensions like code highlighting and snippet tool, etc. to make it fully support Vue. So I would recommend using Atom editor since everything we need is already included in it.

We recommend also installing the Vue Devtools in your browser, allowing you to inspect and debug your Vue applications in a more user-friendly interface.

Our Application

For our application we decided to make a sports media web page. This application allows you to browse articles from different sports, post comments and post new articles.

The application is made up of many different Vue componenets, but there are a few that give the website its functionality. These are the addArticle, showArticles and singleArticle components.

Add Article

This component allows us to add new articles to the website

As you can see the addArticle component has an event that fires when the 'Add Article' button is clicked.

This method sends the article data to the database so that it appears each time the webpage is opended.

Show Articles

In the showArticles component, articles are shown based on which content section you are on. This is done by grabbing the articles from the database, copying then in an array and returning them on the webpage.

Single Article

The singleArticle component shows the user the article that they have clicked on. It also uses the addComment component to allow users to add comments underneath the article.



Conclusion

When working with Vue, you will notice there is a big learning curve that you must get over before you can begin to create your web application. In some cases, learning to use this framework was like learning another coding language. The most challenging part was learning how the entire application runs from a single div on an html page. In Vue, you are creating a single page application that relies on multiple components working together. This can get confusing because one syntax error in a minor component can result in the entire application crashing. Looking back, we would probably take more time to learn the syntax and structure of Vue so that we could better take advantage of its abilities. On the other hand we were glad to take advantage of features like attribute binding, which allows you to dynamically bind and attribute to an expression. This makes things like event handling much quicker by reducing the amount of code written. Another aspect of Vue that we took advantage of is list rendering. While we were creating our app, we worked with lists and arrays constantly. With list rendering, you can loop through an array or list without adding additional javascript (see below):

Overall we enjoyed making our application because it taught us a powerful framework that now gives us the ability to create more advanced web applications in the future. Although we would not recommend the framework for beginners, Vue can be a great tool for intermediate and advanced web developers.
